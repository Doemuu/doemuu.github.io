# Anforderung/Regel Beschreibung
1 Portfolio Die Webseite soll das Portfolio beinhalten.
2 Dokumentation Die Projekte aus dem Portfolio werden korrekt und sauber dokumentiert.
3 Individualität Die Seite muss einen persönlichen Touch haben und soll den Besucher spüren lassen, wer ich bin.
4 Erweiterbarkeit Die Webseite ist dazu gedacht mein Portfolio zu unterstützen, weshalb immer neue Projekte und Reiter entstehen können sollen.
5 Beispiel Die Seite selbst soll ein Vorzeigebeispiel sein.
6 Einfachheit Die Seite wird nicht durch andere Frameworks als JQuery ergänzt
7 Einfachheit Die Seite wird nicht mit anderen Sprachen als HTML, CSS und Javascript ergänzt.

Host

Nachdem ein Klassenkamerad, von der Möglichkeit eine Webseite gratis auf GitHub hosten zu können, erzählt hatte. Entschied ich mich dazu mich in die GitHub Pages einzulesen. Trotz den gewissen Einschränkungen, wie beispielsweise die Verunmöglichung der Verwendung von «PHP», entschied ich mich die überwiegenden Vorteile zu nutzen. Auch die Wartung der Webseite wurde dadurch einfacher. Die Seite wird durch ein einfaches Repository geführt und kann dadurch mit jedem «commit», «push» und «pull request» verändert werden.

Editor

Als Editor der verschiedenen Daten verwende ich den Atom Texteditor. Dieser bringt mit der direkten Verknüpfung zu GitHub und den downloadbaren Packages alle Voraussetzungen mit. Ich verwende für dieses Projekt die Erweiterungspakete «atom-live-server», «atom-file-icons» und den «color-picker». Diese Pakete sind zum grössten Teil «Verschönerungspakete», die die Arbeit im Editor angenehmer gestalten. Doch der Liveserver ist als lokal betriebener Server perfekt für das schnelle und einfache Testen der Applikation geeignet.

Individualität

Um die Individualität und den persönlichen Bezug sicherzustellen, entschied ich mich für die Verwendung eines bestimmten Farbpalette. Mit einem Bot eines Schulkameraden, konnte ich die genauen Farben des Fussball Clubs Luzern Logos ausfindig machen. Diese verwende ich nun als Grundfarben meines Designs.

1.:root {
2. --fcl-yellow: #ffdd00;
3. --fcl-blue-light: #3f79b6;
4. --fcl-grey: #333;
5.}

Erweiterbarkeit

Die Erweiterbarkeit ist durch die Verwendung von einfachen CSS-Klassen in den Reitern Projekte und About Me. Somit kann einfach ein Projekt einer Sprache oder sogar eine neue Sprache hinzugefügt werden. Dasselbe zählt auch für die About Me Attribute.

Redundanzen

Um Redundanzen im HTML Code möglichst gering zu halten, habe ich mich entschieden die Navigationsleisten und den Footer mit HTML include in die index Seiten einzubinden.

Kontaktformular

Das Kontaktformular ist mit dem SMTP von Outlook verbunden und verwendet eine meiner Emailadressen, um auf eine andere Emailadresse die Nachricht zu senden. Um die Sicherheitslücken, die mit der Anmeldung auf meine Emailadresse entstehen, zu stopfen. Benutze ich einen SecureToken. Auch Spam wird gefiltert, da eine tatsächliche Eingabe in den Textfeldern bestehen muss.
